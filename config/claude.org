## TypeScript

### Documentation Strings

- Prefer one-liners if possible
- One-liners start with a capital and don't end with a dot
- Multiple lines are fine but use a single line paragraph for the summary

## Tests

- Use the tested symbol directly for describe summaries
- Start the test summary with `should`
- Structure tests with the AAA pattern ie. `Given`, `When`, `Then`
